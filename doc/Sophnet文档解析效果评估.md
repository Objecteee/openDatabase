## 三、潜在问题与限制

| 问题 | 说明 |
|------|------|
| **输出长度** | 大文件（3.6MB、300+ 页）可能受 API 输出限制，结果在约 1250 行处截断 |
| **表格语义** | 仅文本，无表格单元格边界等结构信息 |
| **版式信息** | 无分栏、分页、字体等版式元数据 |
| **OCR** | 官方说明不支持 OCR，扫描版 PDF 无法解析 |

---

## 四、适用场景评估

| 场景 | 适用性 | 说明 |
|------|--------|------|
| **合同/简历** | ✅ 适合 | 文字型 PDF，表格简单 |
| **统计年鉴** | ⚠️ 部分 | 表格多且复杂时，需后处理 |
| **HSK 大纲类** | ✅ 适合 | 词汇表、目录、多列表格 |
| **扫描件** | ❌ 不适用 | 需 OCR 能力 |
| **图文混排** | ⚠️ 部分 | 仅提取文字，图片内容丢失 |

---

## 五、与业务集成建议

1. **大文件**：可先按页或按章节拆分，再分批调用，避免单次输出过长
2. **后处理**：对表格文本做正则或按列解析，还原为结构化数据
3. **成本**：0.015 PTC/1000 输出字符，大文档需关注用量
4. **格式**：multipart/form-data 上传，与现有文档上传流程兼容

## 七、结论

Sophnet 文档识别 API 对**复杂中文、多页、含表格**的 PDF 解析效果良好：

- 中文、拼音、表格列结构提取完整
- 目录、页码、层级关系基本保留
- 适合作为知识库文档解析的 PDF 处理方案之一

需注意大文件可能的输出截断，以及扫描版需配合 OCR 方案。
