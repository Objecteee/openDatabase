# 切片向量化实现说明

> 文档上传后 → 解析 → 前端本地 embed → 写入 chunks 向量库

---

## 一、流程概述

```
上传完成 (documents.status = pending)
       │
       ▼
用户点击「向量化」（仅 txt、md）
       │
       ├─ GET /api/documents/:id/parse
       │     后端：下载文件 → 解析 → 返回 chunks
       │
       ├─ 前端：embedBatch(chunks)  ← 本地 Transformers.js 模型
       │
       └─ POST /api/documents/:id/chunks
             后端：写入 chunks 表，status → completed
```

---

## 二、已实现模块

| 模块 | 路径 | 职责 |
|------|------|------|
| textParser | `server/src/parsers/textParser.ts` | txt/md 按段落+固定长度切片 |
| parseService | `server/src/services/parseService.ts` | 从 Storage 下载并解析 |
| chunkService | `server/src/services/chunkService.ts` | insertChunks、deleteChunksByDocumentId |
| 路由 | `server/src/routes/documents.ts` | GET `/:id/parse`、POST `/:id/chunks` |
| vectorizeService | `client/src/lib/vectorizeService.ts` | 串联 parse → embed → post |
| DocumentList | `client/src/components/DocumentList.tsx` | 「向量化」按钮 |

## 三、数据库要求

**chunks.embedding** 须为 **vector(384)**（本地模型 paraphrase-multilingual-MiniLM-L12-v2 输出 384 维）。

若当前为 vector(1536)，需在 Supabase SQL Editor 执行：

```sql
TRUNCATE chunks;
ALTER TABLE chunks DROP COLUMN embedding;
ALTER TABLE chunks ADD COLUMN embedding vector(384) NOT NULL;

CREATE OR REPLACE FUNCTION match_chunks(query_embedding vector(384), match_count int DEFAULT 5)
RETURNS TABLE (id uuid, document_id uuid, content text, metadata jsonb)
LANGUAGE plpgsql AS $$
BEGIN
  RETURN QUERY
  SELECT c.id, c.document_id, c.content, c.metadata
  FROM chunks c
  ORDER BY c.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;
```

## 四、使用方式

1. 上传 txt 或 md 文件
2. 文档列表中，待处理行会显示「向量化」按钮
3. 点击后：解析 → 本地 embedding → 写入 chunks，状态变为「已完成」
